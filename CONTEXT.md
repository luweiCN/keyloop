# KeyLoop 训练领域术语 (Context Glossary)

KeyLoop 是一个终端打字训练工具。本文件只定义领域语言，不含实现细节。

## 击键结构 (Keystroke Structure)

打字薄弱的**唯一诊断粒度**。指击键层面的固有属性：**键位**、**手**、**字符转移**（相邻两键的衔接）、**固定符号组合**（编程语言里有限且固定的多字符符号，如 `=>` `!==` `&&` `?.`；**因语言而异**——Rust 的 `::` `->`、Solidity 的 `=>` 等各不相同）、**字符类**（数字 / 符号 / 大小写）、以及**技能维度**（长词、词流畅度等）。

区分"结构"与"内容"的判据：**固定且有限** = 结构（就那几十个键、几十个符号组合，练的是这一段击键序列的肌肉记忆）；**任意且无限** = 内容（任意单词 / 句子 / 文章，其薄弱可再往下分解到结构）。据此 `=>` 是结构（合法靶向单位），`inference` 是内容（不作诊断单位）。

任一字符 / 固定组合的结构属性是其固有特征（`s` 永远属于左手 / Home 行，`=>` 永远是 `=`→`>` 的组合），与它出现在哪个单词、句子、文章里无关。

## 薄弱诊断 (Weakness Diagnosis)

从用户练习的字符级错误与速度，**逐字符**归因到击键结构，判断其薄弱的结构（键 / 手 / 转移 / 固定组合 / 字符类 / 技能维度）。

诊断**跨所有练习形态**：无论错误发生在单词、句子还是文章里，都向下归因到击键结构——内容只是字符流，结构是字符的固有属性，无需"判断这个单词 / 句子里有哪些结构"。

薄弱诊断的产物只用于：决定**练什么结构**（驱动靶向层的加权选材）、以及决定多练哪个**形态**。它**不**以内容单位（词 / 句 / 篇）为诊断单位，也**不**把用户错过的具体内容重新拿来反复练（见 错题回流）。

## 靶向训练层 (Targeted Layer)

针对诊断出的薄弱**击键结构**着重练。两个载体：

- **基础训练**——靶向字母**键位**与**字符转移**；
- **符号 / 数字模块**——同时做**字符靶向**（单个符号 / 数字键）与**结构靶向**（固定符号组合，如 `=>`），并**按编程语言**区分（不同语言的薄弱组合不同）。

粒度是**击键结构**（键 / 转移 / 固定组合 / 字符类），**不含"整词"等内容单位**。"用薄弱诊断指导选材"在这一层正当且核心。

但**靶向 ≠ 重复**：着重练某类薄弱结构，不等于反复发同一份材料。选材应在含该结构的真实素材中**轮换**，**严禁跨天完全重复**（一份数字 / 符号清单两天一模一样是缺陷，不是靶向）。

## 内容练习层 (Content Layer)

单词、句子、文章、代码片段的练习。提供自然、连贯、多样的真实文本，用于综合运用、维持手感，并在单词层顺带学习词汇与读音。

**这一层不回流错题**（不把错过的具体词 / 句重新拿来重练）：内容作为"练习材料 / 学习单位"按自然轮换提供，错几个字符翻篇即过。句子 / 文章 / 代码不按薄弱加权；**单词层例外**——保留按字符类 / 技能维度（大小写、长词）的选材加权（这两个维度的靶向只能在词层体现），但同样不回流具体错词。其余薄弱键 / 转移等结构由靶向层专门强化。

> 单词曾被归入靶向层（"薄弱词优先"），现已移入本层——见 错题回流 与 `docs/adr/0002`。

## 错题回流 (Mistake Replay) — 反模式

把用户曾打错的具体**内容单位**（单词 / 句子 / 段落）重新作为练习材料反复重练。

这被明确认定为**反模式**——"打字不是做题，不搞错题集"。三条理由：

- 打字薄弱是**击键结构**层面的，不是"内容"层面的。用户在某个词 / 句里打错，薄弱的是其中的键 / 转移 / 组合，而非这个词 / 句本身。
- 以内容单位计错有**长度偏置**：长内容天然更容易"被打错"，却不代表它更值得练。
- 结构归因天然按出现频次归一化、可泛化到所有含该结构的材料，信息密度远高于重复单个内容。

因此：**任何内容单位（含单词）都不回流**。薄弱一律通过薄弱诊断在击键结构层面体现，并交由靶向层强化——而靶向层本身也**不重复同一份材料**（见 靶向训练层）。
