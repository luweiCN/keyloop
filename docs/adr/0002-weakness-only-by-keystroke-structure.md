# 2. 薄弱只归因击键结构，内容单位一律不回流

日期：2026-06-20
状态：已接受（推进 ADR-0001）

## 背景

ADR-0001 把训练分为靶向层（键 / 字符 / **词**）与综合应用层（句 / 篇），并废除了句子的整句回流。但它把"词"留在靶向层，保留了**薄弱词回流**：把用户累计打错最多的词优先排进单词练习。

实践暴露三类问题：

1. **薄弱词统计不可信。** 薄弱词取"错误数最多的前 N 个词"，按错误的**绝对数量**计、不除以词长、也无最低次数门槛。后果：20 字母的长词只要错一两个字符就轻易霸榜，3 字母短词几乎无缘——长度偏置严重；一次手滑也可能进榜。这与 CONTEXT.md 当时「薄弱词」定义（"累计多次错"、"多次错门槛"）公然不符。

2. **更根本：词不是打字薄弱的正确单位。** 打字是击键的肌肉记忆，错误的根源在击键结构（某个键、某个手指转移、某类字符、某个固定符号组合），不在"某个词"。同一个薄弱转移会出现在无数词里，重练那一个词只巩固它自己、不泛化。而且任何内容里的错误都能**逐字符**归因到结构层——系统本就在做：字符级维度诊断遍历每次击键、查每个字符的结构属性，不区分形态。于是"词级回流"既劣质又与结构层诊断重复。

3. **内容回流还会"跨天完全重复"。** 即便是合法的结构靶向（符号 / 数字），当前实现也因卡池偏小、去重窗口过短而出现"两天练到一模一样"的内容——这把"针对薄弱"退化成了"重复同一份材料"。

## 决策

把薄弱的针对性**完全收敛到击键结构层**：

- **薄弱诊断只以击键结构为单位**：键位、手、字符转移、固定符号组合、字符类、技能维度。逐字符归因，**跨所有练习形态**（单词 / 句子 / 文章 / 代码里的字符错误都参与）。
- **内容单位（单词、句子、文章、代码片段）一律不作为诊断单位、不回流。** 它们只是"练习材料 / 学习单位"。
- **结构靶向在结构层模块落地**：基础训练靶向字母键 / 转移；符号 / 数字模块同时做字符靶向（单符号 / 数字键）与结构靶向（固定符号组合，如 `=>`），并**按编程语言**区分（不同语言的薄弱组合不同）。两者都按薄弱加权真实材料，且**选材在素材中轮换、严禁跨天完全重复**。**单词、句子、文章、代码等内容层纯自然轮换**，不按薄弱加权、不回流。
- **废弃薄弱词（`focus_words`）**：删除单词模块的薄弱词优先，单词改为从词库自然随机 / 轮换选取（仍保留翻译、朗读等学习属性）。

本 ADR **推进并部分取代 ADR-0001**：其中"靶向层包含单词段、薄弱词优先"的部分作废；靶向层粒度由"键 / 字符 / 词"收缩为纯击键结构（键 / 转移 / 固定组合 / 字符类）。

## 后果

- 根治长度偏置，以及"单词模块前几个每天雷同、连顺序都一样"的体验问题。
- 系统自洽：字符级结构诊断本就跨所有形态、不区分内容；句子 / 文章里暴露的薄弱键继续驱动靶向层。废弃薄弱词后，"薄弱只在结构层、内容层一律自然"完全一致。
- 单词模块降级为内容 / 学习单位：选材自然轮换、新鲜度提升；放弃"词级针对性强化"——这是有意的，针对性交给结构层加权。
- 符号 / 数字模块保留为结构靶向，但需补强：① 薄弱符号 / 组合的诊断**按语言区分**（现 `focus_symbols` 为全局，选材已按语言加载，诊断尚未）；② 扩大卡池 + 拉长去重窗口，**杜绝跨天完全重复**。
- 需要相应改动：移除薄弱词的生成与其全部消费点（单词模块优先选词、依赖薄弱词的模块启用条件、报告页展示等）；其中"某模块是否出现"等条件改用结构层信号或无条件提供。
- 未来读者若看到基础训练 / 符号模块针对薄弱、而单词 / 句子 / 文章都纯自然，不会困惑——本 ADR 即理由。
