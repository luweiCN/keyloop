# 1. 综合应用层不做错题回流，靶向只在低层

日期：2026-06-19
状态：已接受

## 背景

打字训练中，"针对用户薄弱"应该如何、在哪一层体现？

历史实现里有两套机制：

- **薄弱诊断**：从字符级错误与速度，判断薄弱的键、手、字符类、技能维度。
- **错题回流（focus pools）**：把用户错过的具体内容回流为练习材料。其中句子段会把"包含用户错过字符的整句"攒成集合反复重练。

句子段的整句回流暴露出问题：用户打一篇文章时，某句错几个字符是正常的，却因此被反复重练该句——这是"错题集 / 做题"思维，不是打字训练。用户明确反对："打字不是做题"、"我们要检测错误来判断薄弱，而不是搞错题集"。

## 决策

把训练分为两层，薄弱的针对性只作用于低层（见 CONTEXT.md 术语）：

- **靶向训练层**（基础训练、单词段；粒度 = 键 / 字符 / 词）：用薄弱诊断指导选材——薄弱键加权、薄弱词优先。薄弱词需"累计多次错"才算，排除偶然手滑。
- **综合应用层**（句子、文章；粒度 = 句 / 篇）：纯自然、连贯、多样的真实材料，**不靶向特定薄弱点、不回流错题**。

薄弱诊断的产物只用于：决定多练哪个**形态**、为靶向层选材。**不**用于重复用户错过的具体句子 / 段落。

具体地：删除句子段的 `focus.sentences` 整句回流，句子段改为从句子库纯随机选取（不做跨会话去重——刻意避免重现本身又是一种编排，与"自然材料"相悖）。

## 后果

- 句子 / 文章层不再因一次错误反复重练同一句；薄弱键靠自然文本被覆盖到。
- 系统自洽：薄弱诊断本就只加权 keys / words / symbols、从不加权 sentences / articles；删除 `focus.sentences` 后，"靶向在低层、应用层自然"完全一致。
- 句子库需足够大、多样以支撑纯随机（当前约 1500 句）。
- 放弃了"句子级针对性强化"——这是有意的：靶向交给键 / 词层，句子层负责综合运用与维持手感。
- 未来读者看到"基础训练针对薄弱、句子段却纯随机"不会困惑——本 ADR 即理由。
