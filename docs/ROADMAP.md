# KeyLoop 功能规划

[English](ROADMAP.en.md)

KeyLoop 不是通用打字游戏。现阶段它只服务一个人：中文母语、英文普通文本约 50 WPM、代码输入会掉到十几 WPM 的程序员。产品目标是每天打开后直接给出该练的内容，用户不需要自己判断难度、长度、是否练代码。

## 产品原则

1. 软件负责规划，用户负责练习。
2. 默认中文，只保留语言切换这类真实必要的设置。
3. 难度不让用户手动调，由历史数据和后续代码迭代调整。
4. 练习内容必须循序渐进，不能一上来塞代码。
5. 代码练习必须是完整代码块，逐步升级到函数块、组件块、文件片段。
6. TUI 渲染要稳定，切换窗口、回来、退出练习都不能乱屏。

## 目标界面

启动 `keyloop` 后先进入练习菜单：

```text
练习菜单

1. 综合练习  按今日计划从第 1 组练到第 7 组
2. 数据统计  查看总览、历史最好成绩和每天练习明细
3. 热身：基础键位
4. 词块：英文拼写块
5. 高频词：真正英语常用词
6. 单词：前端高频词
7. 专项：数字和符号
8. 命名：大小写和前端 API
9. 代码：短完整代码块
```

选择综合练习后进入今日计划页：

```text
今日练习

1. 热身：基础键位
2. 词块：英文拼写块
3. 高频词：真正英语常用词
4. 单词：前端高频词
5. 专项：数字和符号
6. 命名：大小写和前端 API
7. 代码：短完整代码块

[开始练习]
```

用户不需要选择模式、长度、大小写、数字、符号、代码片段。最多保留：

- 界面语言：中文 / English
- 仓库来源：默认使用内置语料，只有显式传入 `--repo` 才扫描本地仓库
- 入口菜单：综合练习 / 单项练习

语言切换规则：

- 菜单、今日计划、结果和总结页：`L` 切换。
- 跟打页：`Ctrl+L` 切换，普通 `l` 保留给练习输入。
- 课程标题和课程说明跟随当前语言实时渲染，不能固定成中文。

## 练习流程

### 1. 练习菜单

练习菜单只做选择入口：

- 综合练习：按今日计划从第 1 组练到第 7 组。
- 数据统计：展示累计总时长、历史最高 WPM、平均正确率、最低错误率，并支持按日期切换查看当天每次练习。
- 单项练习：只练某一个步骤，例如词块、英语高频词、数字符号或代码块。
- 每日目标用文本显示 `已练 / 目标`，超过目标后继续显示真实累计时长。

### 2. 今日计划页

今日计划由本地历史生成，没有历史时使用固定的新手计划。

显示内容：

- 今天总练习量，例如 15-20 分钟。
- 每组练习的目的，例如热身、符号、词块、代码块。
- 每组预计长度和目标，例如正确率优先、不追速度。
- 只显示“开始”，不显示复杂配置。

### 3. 跟打页

跟打页只做一件事：稳定地敲完当前内容。

要求：

- 文本区域居中，宽度固定，不整屏横铺。
- Ghost text 可读：未输入字符是清楚的浅色，不用暗灰。
- 正确字符变绿，错误字符变红并带下划线。
- 中文输入法提交的非 ASCII 字符忽略，不推进练习。
- 切换窗口回来后强制重绘，避免界面乱。
- 完成当前组后停住，显示结果和下一组入口。

### 4. 结果页

每组完成后显示：

- WPM / 原始 WPM
- 正确率
- 错误数
- 退格数
- 慢词块
- 错误字符 / 错误符号
- 下一组是什么

用户可以：

- Enter：进入下一组
- R：重练当前组
- Esc/Q：保存并退出

## 内容系统

后续主要工作中心不是增加设置，而是持续丰富课程内容：

- 扩充基础键位和字母过渡材料。
- 扩充英文拼写词块，例如 `the`、`tion`、`ing`、`ment`、`pre`、`con`、`str`。
- 扩充真正英语高频词，不涉及大小写。
- 扩充程序员高频词、前端变量名。
- 扩充数字、符号、括号、引号、箭头、比较符专项。
- 扩充 TS / JS / Solidity / HTML / CSS / Less / Sass 的完整代码块。
- 从本机仓库中建立代码块库，按语言、难度、来源分类。

### 阶段 1：基础输入和英文词块

目的：先把键位、英文拼写块、常见字母组合打稳。

内容：

- home row 和常用过渡：`asdf`、`jkl;`、`qwer`、`uiop`
- 英文拼写块：`the`、`tion`、`ing`、`ment`、`pre`、`con`、`str`
- 真正英语高频词：`the`、`people`、`because`、`through`
- 程序员和前端高频词：`return`、`current`、`response`
- 不包含真实代码

进入条件：默认第一阶段。

升级条件：连续多天正确率稳定在 95% 以上，退格不过高。

### 阶段 2：数字和符号

目的：解决代码输入明显掉速的主要来源。

内容：

- 数字行：`0-9`、常见状态码、数组索引
- 符号：`() [] {} <>`、`=>`、`!==`、`&&`、`||`
- 短组合：`() => {}`、`items[0]`、`value !== null`

进入条件：基础输入稳定，或者历史记录显示符号错误很多。

### 阶段 3：命名和大小写

目的：适应真实编程命名，而不是手动开“大小写练习”。

内容：

- camelCase：`selectedType`、`currentValue`
- PascalCase：`SortingState`、`HTMLInputElement`
- 常见前端词：`useEffect`、`onChange`、`preventDefault`

进入条件：词块和符号不再是主要瓶颈。

### 阶段 4：短完整代码块

目的：开始练代码，但只练完整、短小、可读的代码块。

内容示例：

```ts
const result = items.map((item) => item.id);
```

```ts
if (value !== null && value !== "") {
  return value;
}
```

要求：

- 不能只抽单行碎片。
- 优先完整语句、完整 if 块、完整 callback。
- 每个块要有清晰边界。
- 难度由长度、符号密度、换行数、历史错误决定。

### 阶段 5：函数块 / 组件块

目的：练真实代码结构。

内容：

- 完整函数
- React hook 片段
- 事件处理函数
- 小组件

要求：

- 代码块可以语法高亮。
- 未输入部分仍然保持 ghost text 效果。
- 代码块提取要尽量基于语法树，至少先用括号/缩进匹配，不再只按行抽。

### 阶段 6：文件片段

目的：接近真实工作输入。

内容：

- 文件中的连续片段
- 最近修改过的代码
- 当前项目高频模式

进入条件：函数块阶段稳定后再开启。

## 自动计划逻辑

每天计划由最近练习记录生成：

1. 如果没有历史：固定第 1 周新手计划。
2. 如果正确率低于 95%：减少长度，优先基础和专项。
3. 如果退格很多：降低速度要求，重复慢词块。
4. 如果符号错误多：加入符号专项。
5. 如果词块慢：加入高频词块专项。
6. 如果基础稳定：加入短代码块。
7. 如果短代码块稳定：升级到函数块。

软件内不暴露“难度选择”。我之后可以根据一周结果直接改代码和计划规则。

## 代码块提取方向

当前实现偏简单，容易抽到单行碎片。后续要改成完整代码块优先：

1. 第一版：基于缩进和括号匹配提取完整块。
2. 第二版：按语言引入 parser，例如 tree-sitter。
3. 第三版：给代码块做难度评分和历史匹配。
4. 第四版：扫描本机常用代码目录，建立可复用的本地代码块索引。

优先级：

- 完整 `if` / `for` / `while` / callback 块
- 完整函数
- 完整 React 组件或 hook
- 文件连续片段

过滤：

- 跳过 lock 文件、生成文件、压缩文件。
- 跳过过长代码块。
- 跳过注释和无意义配置。
- 初期不选太复杂的泛型、正则、长链式调用。

本机仓库索引不应该每次启动全量扫描。更合理的方式是：

- 首次运行或手动命令扫描用户指定的常用代码目录。
- 把提取出的代码块缓存到本地。
- 练习时直接从缓存里按语言和难度选择。
- 后续再支持“最近修改过的仓库/文件”优先。

## 渲染稳定性

必须修：

- 每帧绘制前清屏或完整覆盖当前区域。
- 终端 resize 后重新计算布局并重绘。
- 切换窗口回来后不要残留旧画面。
- 宽终端下内容区仍然居中。
- 小终端下显示清楚的提示，不画坏框。

## 实现阶段

### P0：砍掉无效设置

- 设置页改成“今日计划页”。
- 删除 TUI 里的练习类型、长度、大小写、数字、符号、代码片段开关。
- 只保留语言切换。
- Enter 直接开始今天第一组。

### P1：多组练习流程

- 今日计划生成多个练习组。
- 完成一组后进入结果页，再进入下一组。
- 全部完成后显示今日总结。
- 每组单独保存结果，并标记所属计划。

### P2：内容生成器重做

- 把内容按阶段组织，而不是按用户开关拼接。
- 基础、词块、数字、符号、命名、短代码块分别有清晰边界。
- 默认计划从基础和专项开始，不主动上真实代码。

### P3：完整代码块提取

- 改掉单行优先策略。
- 先做括号/缩进块提取。
- 后续再接 tree-sitter。

### P4：历史分析和周迭代

- `keyloop report today` 输出当天总结。
- `keyloop report week` 输出一周复盘。
- 我根据一周数据判断是否升级阶段，并直接调整代码计划。
