# 跟打区域翻译/释义对齐显示设计

日期：2026-06-10
范围：`ts/src/ui/opentui/renderer.ts`（仅渲染层），`ts/tests/opentuiRenderer.test.ts`

## 背景与问题

日常练习的跟打文本区域（`renderGhostText`）目前对三类标注（`PracticeTargetAnnotation.display`）的呈现方式各不相同，且对应关系不清晰：

1. **单词（`display: "word"`）**：单词行下方的释义行用固定 18 列宽并排堆放（`renderGhostMeaningLine`），与上方单词的位置没有对齐关系，看不出哪条释义对应哪个单词。
2. **句子（`display: "line"`）**：翻译不在跟打区域内，而是堆放在跟打框下方的「译 …」列表（`renderLineTranslations`），与句子没有一一对应。
3. **文章（`display: "article"`）**：渲染在跟打框下方一个带边框、带「全文翻译」标题的独立区域（`renderArticleTranslation`），固定高度 3 行且截断为单行，看不全。

## 目标

统一为：**所有翻译/释义都是跟打区域内的暗色（`theme.muted`）辅助文字，显示在对应原文的正下方**。全部在渲染层实现，跟打文本本身一个字符都不变，用户需要键入的内容与现在完全一致。

## 设计

### 1. 单词释义列对齐（`display: "word"`）

- 保持一行多个单词的目标文本布局不变。
- 释义行不再使用 flex 固定宽度堆放，改为按列位置渲染：根据每条 annotation 的 `start` 偏移计算单词在源行内的列位置，把释义绘制在对应单词正下方的同一列起点。
- 中文释义按显示宽度计算（全角字符 = 2 列）。
- 若某条释义过长、会撞到下一个单词的列起点，则截断并保留至少 1 格间隔，保证后续释义仍然对齐。
- 单词行由每行词数（`perLine`）控制，通常不折行；若发生折行，释义按 annotation 所落在的视觉行分别对齐。

示例：

```
apple        banana       cherry
苹果         香蕉         樱桃
dog          elephant     forest
狗           大象         森林
```

### 2. 句子翻译（`display: "line"`）

- 删除跟打框下方的「译 …」堆放列表（`renderLineTranslations`）。
- 改为在跟打框内、每个句子所在源行的最后一个视觉行**下方**插入一行暗色翻译，做到「一句原文 + 一句翻译」上下相邻。
- 翻译行超宽时截断（与现有单行行为一致）。

### 3. 文章整段翻译（`display: "article"`）

- 删除现有带边框、带「全文翻译」标题的独立区域（`renderArticleTranslation`）。
- 改为在跟打框内、文章正文末尾的下方，用暗色文字渲染整段翻译，自动换行完整显示，不加边框、不加标题。
- 不做句子级对应——整篇文章对应一段完整翻译。

### 混合练习

同一目标内同时存在 word 与 line 标注时（如 everyday mix），两种标注各自生效：单词块每行下方有对齐的释义行，句子块每句下方有翻译行，互不冲突。

### 不改动的部分

- `targets.ts` 内容生成逻辑、annotation 数据结构均不变。
- 跟打输入、判分、折行（`wrapGhostRows`）逻辑不变。

## 测试

在 `ts/tests/opentuiRenderer.test.ts` 增加：

1. 单词目标：释义行中每条释义的列起点与对应单词的列起点一致；过长释义被截断不撞下一列。
2. 句子目标：每个句子行的下方紧跟一行对应翻译；跟打框下方不再渲染「译 …」列表。
3. 文章目标：正文下方渲染完整多行翻译；不再渲染带边框的「全文翻译」区域。
