# 单词发音与重复次数统一

日期：2026-06-12
状态：待用户确认

## 背景与目标

当前单词类练习已经有部分重复次数能力，但覆盖不完整：日常单词、编程常用词、技术长词已接入，**自建词库单词**还没有接入。用户还希望所有单词相关练习都能在输入时读一遍单词，并且有一个默认关闭的总开关，进入练习后可以用 `Ctrl+O` 临时调整。

本次目标：

1. 把「单词重复次数」补齐到自建词库单词。
2. 把「单词发音」接入所有单词流：日常单词、编程常用词、技术长词、自建词库单词。
3. 发音默认关闭；全局设置决定默认值；练习中 `Ctrl+O` 可调整当前练习。
4. 发音只按单词读，不读释义，不读整句/整篇。

## 覆盖范围

接入的目标：

| 模块 | 内容形态 | 重复次数 | 发音策略 |
| --- | --- | --- | --- |
| 日常练习 / 单词 | 普通英文单词 | 已有，保持 | 免费词典音频优先，未命中再调有道 |
| 编程基础 / 编程常用词 | 编程语境英文词 | 已有，保持 | 直接调有道 |
| 编程基础 / 技术长词 | 技术长词 | 已有，保持 | 直接调有道 |
| 自建词库 / 单词 | 用户自建英文单词 | 新增 | 免费词典音频优先，未命中再调有道 |

不接入的目标：

- 自建词库词组、句子、文章。
- 日常词组、句子、文章。
- 代码、符号、API 调用、命名形式等非单词流。

原因：本次功能是「单词读音」和「单词重复肌肉记忆」，不是整段 TTS。词组/句子/文章后续可以单独设计朗读规则、分段缓存和播放节奏。

## 发音服务决策

### 免费词典音频

日常单词和自建词库单词优先查 `dictionaryapi.dev`：

- 免费、无需 key；
- 接口返回 `phonetics[].audio`；
- 覆盖日常词较好，但技术词覆盖不稳定。

官方文档：https://dictionaryapi.dev/

### 有道 TTS

编程常用词和技术长词直接走有道；日常单词和自建词库单词在免费词典没有音频时兜底走有道。

有道官方约束：

- 文本参数 `q` 的 UTF-8 编码长度上限是 2048 字节；
- 在线 TTS 以成功调用次数计费；
- 新账号有 50 元试用额度；
- 低用量档公开价为 0.012 元/次。

官方文档：

- TTS 接口：https://ai.youdao.com/DOCSIRMA/html/tts/api/yyhc/index.html
- 价格说明：https://ai.youdao.com/DOCSIRMA/html/tts/price/yyhemxdy/index.html

### 有道凭据

凭据不写进仓库，也不写进练习记录。运行时从环境变量读取：

```text
YOUDAO_APP_KEY
YOUDAO_APP_SECRET
```

如果用户开启发音但没有配置有道凭据：

- 免费词典命中时正常播放；
- 需要有道兜底的单词静默跳过，不阻塞输入；
- 设置页显示「有道未配置」状态。

## 设置模型

新增全局偏好：

```typescript
interface WordAudioPreferences {
  enabled: boolean;      // 默认 false
}
```

含义：

- `enabled=false`：所有单词练习默认不发音。
- `enabled=true`：所有支持发音的单词练习默认发音。
- 进入练习后，`Ctrl+O` 的「发音」开关只影响当前练习；它的初始值来自全局开关。

自建词库单词新增重复次数偏好：

```typescript
interface CustomLibraryPreferences {
  word_repeats: number;  // 1-10，默认 1
}
```

已有模块的重复次数规则保持一致：范围 `1-10`，默认值沿用当前实现。

## 练习选项面板

`Ctrl+O` 在以下练习中显示两个与单词相关的选项：

- `单词重复`：`1` 到 `10`。
- `发音`：开 / 关。

覆盖：

- 日常单词；
- 编程常用词；
- 技术长词；
- 自建词库单词。

重复次数改变后，当前练习重新生成 target，沿用现有即时生效逻辑。发音开关改变后，不重新生成文本，只改变后续单词是否播放。

## 重复次数排版规则

自建词库单词复用当前已确认的单词排版规则：

1. 单词重复 N 次时，只重复英文，不重复中文释义。
2. 重复后的英文整体与中文释义对齐。
3. 重复次数为 1 时，沿用日常单词的竖向对齐规则。
4. 重复次数大于 1 时，使用松散排版：只按当前行能放下多少个完整单词组来换行，不再做跨行竖向对齐。
5. 换行只能发生在单词组之间，不能拆开单词，也不能把中文释义插到重复英文中间；如果一个重复组跨多行，中文释义显示在该组所有英文行之后。

## 发音触发规则

发音的单位是「原始单词」，不是重复后的文本。

示例：目标文本为

```text
availability availability availability
有效性
```

播放内容只应是：

```text
availability
```

触发时机：

1. 用户开始输入某个单词时，播放该单词一次。
2. 用户用空格进入下一个单词时，播放下一个单词一次。
3. 同一个单词重复多遍时，不因重复文本播放多次。
4. 中文释义不播放。
5. 回车不作为单词跳转键；单词之间仍以空格推进，保持当前统一体验。

为避免从重复文本里猜原词，`PracticeTargetAnnotation` 增加可选字段：

```typescript
interface PracticeTargetAnnotation {
  start: number;
  end: number;
  translation_zh?: string;
  display?: "word" | "word_loose" | "phrase";
  audio_text?: string;
}
```

单词 target builder 负责写入 `audio_text`。播放层只读 `audio_text`，没有该字段则不发音。

## 音频缓存与播放

新增音频服务：

```text
src/audio/wordAudio.ts
```

职责：

1. 根据模块选择 provider 链。
2. 下载音频并缓存到本地。
3. 非阻塞播放缓存文件。
4. 失败时静默降级，不影响输入。

缓存目录：

```text
~/.keyloop/audio-cache/
```

缓存 key 包含：

- provider；
- 规范化后的单词；
- voice 参数；
- API 版本。

播放策略：

- macOS MVP 使用系统 `afplay` 播放本地音频文件；
- 播放异步执行，不阻塞按键处理；
- 一个练习中同一个 annotation 只触发一次播放；
- 同一个单词多次出现时复用缓存，不重复调用 API。

## 实现切点

1. 偏好设置：新增 `word_audio.enabled`，新增/补齐 `custom_library.word_repeats`。
2. 自建词库单词 target：支持重复次数、`display: "word"` / `"word_loose"`、`audio_text`。
3. 已有单词 target：为日常单词、编程常用词、技术长词补 `audio_text`。
4. 练习选项：把 `单词重复` 和 `发音` 接到四类单词 source。
5. 音频服务：实现 provider 链、Youdao 签名、缓存、异步播放。
6. 输入流：在 active annotation 变化时触发一次 `audio_text` 播放。
7. 设置页：显示全局「单词发音」开关和有道配置状态。

## 测试策略

单元测试：

- 自建词库单词重复次数：`1` 与 `>1` 的文本、释义、`display`、`audio_text`。
- 四类单词 target 都带正确 `audio_text`。
- `Ctrl+O` 选项覆盖四类单词 source。
- 发音 provider 路由：日常/自建先免费后有道，编程常用词/技术长词直接有道。
- Youdao 签名和 2048 字节限制处理。
- 缓存命中时不重复请求 provider。
- 输入推进到新 annotation 时只播放一次；重复单词不多次播放；中文释义不播放。

## 不做的事

- 不做整句、整段、整篇文章朗读。
- 不在界面里录入或保存有道密钥。
- 不把回车改成单词跳转键。
- 不新增词组发音；词组发音需要单独考虑播放粒度。
- 不为 Linux/Windows 播放器做完整适配；MVP 先保证当前 macOS 环境。
